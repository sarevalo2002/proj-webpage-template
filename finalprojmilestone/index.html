What We've Accomplished and Preliminary Results

We decided to go with a fluid simulation modeled from a paper called Position Based Fluids by Macklin, Muller. In this paper, the fluid simulation is based on many spherical particle collisions which together will emulate the behavior of a fluid; when combined with a shader, it creates quite a realistic effect of water, and the paper allows includes a few formulas to test concepts such as viscosity and density, so we believe we are going in the right design direction.

To accomplish this, we first implemented a fluid class which contains a collection of particle objects. Under the hood, these particle objects are actually using the sphere.cpp’s mesh generation in order to be rendered. However, they have a complete separate set of properties different from sphere.c which are similar to the point masses’ implementation - particles only use the sphere’s mesh to be rendered. Then, we had these particles simulated to fall due to gravity with ClothSim’s Verlet Integration, and we were able to achieve a rendering of this displayed below. The main idea we are trying to achieve is to have the fluid particles fall into a box and splash around, so we built a box consisting of rectangles which was also able to be rendered. These rectangles were constructed by truncated planes which had boundaries defined in the respective JSON files, which is also where the fluid’s parameters are specified.

In the end, our progress so far has allowed us to simulate and render fluids as a collection of spheres subject to external forces. As shown in our demo, we display spheres falling as a result of gravity, in the same vein as how the cloth interacts with the world in plane.json.
